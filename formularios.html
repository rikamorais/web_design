Tag <form>...</form>

As tags para a criação de formulário são <form> e </form>

Os atributos da tag <form> são:

  • action: informa o endereço (URL) para o qual o conteúdo do formulário será enviado. Se este atributo for omitido, o endereço corrente (página atual) será usado.
  • method: informa o método de transferência dos dados que será usado pelo atributo “action”, sendo possível adotar os valores:
  • get: valor padrão do atributo “action”, usado quando se deseja enviar os dados de um formulário para um endereço (URL de um servidor), seguindo o formato “endereço?nome=ação”. Esse tipo de ação apresenta os dados do formulário na barra de endereços do browser.
  • post: faz com que os dados do formulário sejam enviados para o servidor definido no atributo “action” em um bloco de dados. Este é o método mais indicado, uma vez que os dados não são apresentados na barra de endereço do navegador.
  • enctype: especifica a forma de submissão dos dados de um formulário quando se usa o valor “post” no atributo “method”. O valor padrão desse atributo é “application/xwww- form-urlencoded” (dados do formulário são passados de forma codificada). Caso seja usado com o valor “file” do atributo “type” (será mostrado mais adiante), o valor desse atributo deve ser “multpart/form-data” (dados do formulário são passados em multipartes). É muito comum o uso do valor “text/plain” para envio de dados de um formulário de maneira formatada.
  • autocomplete: (criado no HTML 5) indica que o browser pode (conteúdo = “on”) ou não (conteúdo = “off”) recuperar automaticamente uma informação já preenchida anteriormente pelo usuário (exemplo: nome, endereço, email etc.) Normalmente este atributo é usado com elemento input.


Tag <fieldset>...</fieldset>

é usada para agrupar elementos relacionados em um formulário. A tag desenha uma caixa ao redor dos elementos relacionados.



Tag <legend>

A tag define uma legenda para o elemento <fieldset> .

Exemplo:

                <form>
                    <fieldset>
                        <legend>Exemplo
                        Legenda
                    </fieldset>
                </form>

Legenda



Tag <label>

A tag <label> apresenta um texto de identificação (rótulo). Está ligado diretamente com outros elemento de um formulário e sua função é identificá-los. Possui os seguintes atributos:

  • for: o nome interno associado ao atributo “id” de identificação de outros elementos de formulário (serão apresentados em seguida).
  • title: o texto explicativo que será apresentado quando o ponteiro do mouse permanecer sobre esse elemento por mais de um segundo. Esse atributo é opcional.

Exemplo 1 de uso: <label>Nome</label>

Exemplo 2 de uso: <label for=“cep”>CEP</label>

Exemplo 3 de uso: <label title=“Exemplo de observação”>OBS.:</label>

Ainda assim, o uso de um label não consegue ser evidentemente demonstrado sem estar associado a algum outro elemento. Veja a seguir um uso mais eficiente dele.

Atenção: Para esse elemento é necessário abrir e fechar a tag: <label>...<label>



Tag input

(tag órfã - <input/>): utilizado para criar campos de entrada de dados em um formulário. É o recurso mais comum e mais utilizado. Permite a inserção de múltiplos componentes de controle em um formulário, que podem ser campos de textos, botões, controles deslizantes e outros. Possui os seguintes atributos:

  • name: estabelece o nome do elemento (campo, área de texto ou botão) no formulário. Obrigatório quando há mais de um elemento do mesmo tipo.
  • id: nome de identificação associado ao atributo “for” da tag <label>.
  • type: tipo de entrada de dados que o campo vai receber. Aceita os valores:
  • text: formato caixa de texto, utilizado para receber dados do tipo caractere (texto simples).
  • password: utilizado exclusivamente para campos de senha.
  • radio: tem a forma de um botão circular, utilizado para selecionar apenas uma opção numa lista de opções predefinidas (todas as opções devem ter o mesmo conteúdo no atributo name).
